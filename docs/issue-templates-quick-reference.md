# Issue Templates Quick Reference

A quick guide to help you choose the right issue template.

## Template Selection Guide

| If you need to... | Use this template |
|-------------------|-------------------|
| Report something broken or not working | **Bug Report** 🐛 |
| Request a new feature or capability | **Feature Request** ✨ |
| Change colors, layout, or visual design | **UI/Styling Enhancement** 🎨 |
| Update README, docs, or comments | **Documentation Update** 📝 |
| Add/modify/remove an activity | **Activity Management** 📅 |

## Required Information by Template

### Bug Report 🐛
- ✅ Problem description
- ✅ Current behavior
- ✅ Expected behavior
- ✅ Location of bug
- ✅ Acceptance criteria

### Feature Request ✨
- ✅ Feature description
- ✅ Problem or need
- ✅ Proposed solution
- ✅ Acceptance criteria
- ✅ User benefit
- ✅ Priority level

### UI/Styling Enhancement 🎨
- ✅ Styling request
- ✅ UI area affected
- ✅ Current state
- ✅ Desired state
- ✅ Acceptance criteria
- ✅ Urgency level

### Documentation Update 📝
- ✅ Documentation issue
- ✅ Documentation type
- ✅ Location
- ✅ Current content issues
- ✅ Desired content
- ✅ Acceptance criteria
- ✅ Priority level

### Activity Management 📅
- ✅ Request type (add/modify/remove)
- ✅ Activity name
- ✅ Activity details
- ✅ Category
- ✅ Schedule information
- ✅ Maximum participants
- ✅ Reason for change
- ✅ Acceptance criteria
- ✅ Effective date

## Common Mistakes to Avoid

❌ **Vague descriptions**
- Bad: "The website doesn't look good"
- Good: "The website uses blue colors, but our school colors are white and lime green"

❌ **Missing acceptance criteria**
- Bad: "Fix the colors"
- Good: 
  - [ ] Header uses lime green (#32CD32)
  - [ ] Cards have white background
  - [ ] Text is readable

❌ **Incomplete information**
- Bad: "Add new activity"
- Good: "Add Robotics Club - Technology category, Tuesdays/Thursdays 3-5pm, max 20 students"

❌ **Multiple unrelated requests**
- Bad: "Fix colors AND add calendar view AND update docs"
- Good: Create separate issues for each request

## Tips for Success

1. **Search first** - Make sure someone hasn't already submitted a similar issue
2. **Be specific** - Include exact colors, times, names, etc.
3. **Use examples** - Show reference images or links when helpful
4. **Test first** - Verify the issue actually exists
5. **One issue = one problem** - Don't combine unrelated requests

## Need Help?

- Not sure which template? Choose the closest match and explain in Additional Context
- Missing information? Add what you know and mention what you're unsure about
- Technical terms? Describe it in plain language - that works too!
- Still stuck? Use GitHub Discussions for general questions

## Template Features

All templates include:
- **Guided fields** to ensure you provide necessary information
- **Dropdown menus** for common options
- **Required validation** on critical fields
- **Examples and placeholders** to guide you
- **Acceptance criteria section** for clear success metrics
- **Additional context** for anything else relevant
